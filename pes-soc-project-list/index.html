PESOS SoC Management Portal


PES SoC will be an annual event and a lot of people can get to do projects every summer. To be able to manage the applicants, the mentors and to evaluate the project, we need a portal where all the information can be exchanged.
The idea is simple, create a website similar to GSoC to manage PES-SoC projects. There is no restriction on the technologies that you can use to build the portal. The following are the minimum requirements -

  • A page for students and mentors to register.
  • A list of projects that a student can browse through.
  • A mentor should be able to add new projects and edit descriptions and see the applicants and their proposals.
  • A WYSIWYG editor to submit proposals and a comments section/issue tracker for interaction among the contributors.
  • A local PES-Git repository where all the code will be hosted.

The portal will be hosted locally and we can get the college to provide external access. The community will take care of getting the necessary permissions to provide access and to get a hosting service after the project is complete. Melange itself is open source and if you are planning to use Melange, then we’d like to see some more features. Bonus requirements -

  • Make the whole PES SoC process social. Let the commits appear on a common timeline, or people can follow each other and you can see their commits and posts.
  • Gamify the coding process and make it more fun for everyone involved.
  • Integrate Facebook, Twitter etc.
  • Add a wiki and a journal for each accepted project. The wiki will be used to give information about the project and the journal is a blog where the team can write about their experience.
  • Anything innovative that you can think of!

 

Prerequisites:

  • HTML, CSS, JavaScript for frontend development.
  • Backend – php / python (django) / Node or some CMS.
  • A DBMS – MySQL, or any NoSQL database. (I think NoSQL will be a bad idea here)
  • A strong motivation and interest to complete the project in 10-12 weeks.

You can visit www.w3schools.org to brush up your web skills.
Some links that might help you to write your proposal -
Django – https://www.djangoproject.com/
Melange – http://code.google.com/p/soc/wiki/MelangeIntro
Joomla – http://www.joomla.org/
Drupal – http://drupal.org/
Mentor: Akshay M S – (akshayms91 AT gmail DOT com)
Team Size: Upto 3



Attendance Tracker


We already have a system in our college where lecturers take attendances on a mobile device, which is updated to the college’s database immediately. If taken on register, it is updated once a week or so. The project involves having a place where the student can get all the statistics regarding the classes he/she missed, whether it’s falling in the shortage zone etc. Updates for him/her once a fortnight or so. Using technologies like textweb, we can also write scripts that alert a student once the percentage of attendances falls under the requirements so that he/she can be more careful in the future (Addition of more features left to applicant’s creativity. More the features, better the chances of selection). Since SoC runs for three months, the SoC student would have to implement this as both web app and mobile app.

Expected results: A web app and mobile app/s that allow a student to track his attendance statistics at any time, with additional features like attendance shortage alert etc. (More weightage for addition of more innovative ideas)

Prerequisites:
For web app: Basic HTML5, css, javascript , one of the server side languages like python, php, ruby etc. (additional skills like jQuery, django, RoR etc. are a bonus)

For mobile app: Working knowledge on any platform of choice (iOS, Android, Qt, j2ME) If team size is more than two, propose to implement in at least two platforms

Mentor: Navaneeth Bhat (navneeth DOT neo AT gmail DOT com)
Co-mentor: Shrikrishna Holla (shrikrishna DOT holla AT gmail DOT com)

Note: This project is open only for second and fourth semester students.

CIE Marks' Automation


Brief explanation: Currently how the process works in our college
Professors evaluate our Blue books, then show it to the students, any modifications to it are directly made in the Blue Book. The Blue books are later sent physically to the respective Department Office. The Dept office staff will later enter each marks from the Blue Book into an Excel Sheet manually for every course of every student of every semester in their respective departments.
This is surely a hell lot of unwanted work. The whole process can be automated by

  • Addition/Modification/Deletion of marks online.
  • Automated generation of CIE marks sheet which will later be posted to the home address of the students.
  • An option to import these information from the Database as an Excel Document.
  • Role based access permission rules.
    Example – A Data Communication teacher should be allowed to change only the marks or related stuff only for that particular subject
  • The office staff should have read-only access to all the information posted by all the teachers, because they are supposed to use the data, not modify it.

Expected results: A secure, complete, useful tool (or rather you may want to call it a product) which can be deployed in each department. Next semester it will run in alpha in all the departments because there may have to go a lot of bug fixes, and from the coming semesters this will be the new marks maintaining tool.
Knowledge Prerequisite: DBMS, HTML, CSS, JQuery, PHP(You can learn them all after you start the project.)
Mentor: Vivek P (vivek DOT purushoth AT gmail DOT com)
